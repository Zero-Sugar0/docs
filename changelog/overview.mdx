<Update label="June 30, 2026" tags={["v1.10.4"]}>
  ## v1.10.4 — ZilMate Ubiquity: system-wide AI hotkey

  ### New features

  - **ZilMate Ubiquity background service.** Highlight text in any app, press <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>Z</kbd> on Windows or <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>Z</kbd> on macOS, and ZilMate rewrites, translates, or answers in place — without leaving your current window. Your original clipboard is backed up and restored automatically. See [Ubiquity overview](/ubiquity/overview).
  - **Lightweight, local-only daemon.** Ubiquity runs entirely on your machine with no extra runtimes to install. Requests stay on the loopback interface and are protected by a per-machine token, so no other app on your network can talk to it.
  - **Pure Output Mode responses.** Selections are sent through ZilMate in a mode tuned for inline replacement — no preambles, no follow-up questions, just the answer ready to paste.

  ### Updates

  - **New `zilmate ubiquity` commands.** Install, start, stop, check status, and uninstall the Ubiquity service from the [CLI](/cli/overview): `zilmate ubiquity install`, `start`, `stop`, `status`, and `uninstall`.
</Update>

<Update label="June 28, 2026" tags={["v1.10.3"]}>
  ## v1.10.3 — Smarter self-optimization and steadier prompts

  ### New features

  - **Coding agents now read and write the Corporate Wiki.** Internal app-builder and QA agents can pull live requirements, schemas, and monetization guides from the shared [Corporate Wiki](/memory/corporate-wiki) before scaffolding code, and publish what they learn back for the rest of the swarm. Code generation stays in sync with the rest of the Digital Corporation.

  ### Updates

  - **Offensive + defensive optimization loop.** The background self-learning loop now captures both defensive guardrails (preventing failures) and offensive accelerators (reusable patterns, prompt enhancements). Expect richer recommendations as your swarm runs more tasks.
  - **Semantic rule deduplication.** The optimizer first checks existing rules in the wiki and only emits net-new or genuinely updated guidelines, so your knowledge base stays tight instead of accumulating duplicates.

  ### Bug fixes

  - **Cleaner interactive prompts.** The "thinking" ticker no longer overwrites confirmation prompts or multi-select checkboxes in the [CLI](/cli/overview). Background spinners pause while you're answering a prompt and resume when you're done — no more misaligned lines or corrupted selections.
</Update>

<Update label="June 28, 2026" tags={["v1.10.2"]}>
  ## v1.10.2 — Cleaner start-up and a roomier trace viewer

  ### New features

  - **Interactive swarm trace diagnostics.** The HTML trace report now ships with collapsible directory tree nodes (with auto-expand on query match), clickable KPI metric cards that filter traces in place, a chronological Gantt timeline with dotted tracking guides, and a one-click **Copy JSON** utility for any inspector panel. See [Swarm overview](/swarm/overview).

  ### Updates

  - **Pristine CLI start-up.** Running any `zilmate` command now opens straight to the ASCII welcome banner — no more `dotenv` injection counts or tip banners cluttering the first few lines. Environment loading is unchanged: local overrides still take precedence. Try it from any command in the [CLI reference](/cli/overview).
  - **Fluid-grid trace layout.** Trace timelines, Gantt columns, and pre-formatted output now scale with screen height instead of being capped by fixed viewport limits, so large traces render edge-to-edge on big monitors.
  - **Expanded inspector panels.** Raw JSON, [collaboration reports](/swarm/joint-war-rooms), and [wiki cards](/memory/corporate-wiki) no longer sit inside short scroll boxes — long outputs render in full as part of a single, natural page scroll.

  ### Bug fixes

  - Removed nested scrollbars in the trace inspector that made it hard to read or copy large payloads.
  - Suppressed stray configuration output that occasionally printed above the start-up banner.
</Update>
