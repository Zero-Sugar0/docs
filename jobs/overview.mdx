ZilMate's job system gives any agent task a durable, retryable execution lifecycle — schedule work by cron, run it on demand, and retrieve logs when it finishes. Whether you need a weekly GitHub summary, a nightly analysis run, or an on-demand data digest, the job system queues and tracks every execution from creation through completion.

## Job lifecycle

Every job moves through a fixed sequence of states:

<Steps>
  <Step title="Queued">
    You call `createJob()`. ZilMate writes the job record and assigns it a unique ID. Its initial status is `queued`.
  </Step>
  <Step title="Running">
    The local worker or a QStash webhook delivery picks up the job and begins executing the task. Status transitions to `running`.
  </Step>
  <Step title="Succeeded or Failed">
    ZilMate writes the final status (`succeeded` or `failed`) and appends a log entry. If the job fails and `maxAttempts` allows, it re-queues for a retry.
  </Step>
</Steps>

## Storage

ZilMate persists all job records in a SQLite database named **`zilo-jobs.db`**, located inside your `ZILMATE_WORKSPACE` directory. This means jobs survive process restarts and are available for inspection or replay at any time.

## Execution modes

ZilMate supports two ways to execute due jobs:

**1. Local worker** — Run `zilmate jobs worker` from your terminal. The worker polls for due jobs and executes them on your machine while the process is active. This is the fastest path during development.

**2. QStash** — For hosted or serverless environments, register your job's cron schedule with Upstash QStash. QStash calls your public webhook URL on schedule, so jobs execute even when your local machine is offline.

<Note>
  Set `ZILMATE_JOBS_ENABLED=true` in your environment to activate the job system before using either execution mode.
</Note>

## SDK methods

Use the following methods on a `createZilMate()` instance to manage jobs programmatically.

### `createJob(options)`

Creates a new background job and persists it to SQLite.

<ParamField body="task" type="string" required>
  The natural-language instruction the agent will execute when the job runs.
</ParamField>

<ParamField body="schedule" type="string">
  A standard 5-field cron expression (e.g. `0 9 * * 1`). Omit for on-demand-only jobs.
</ParamField>

<ParamField body="maxAttempts" type="number">
  Maximum number of execution attempts before the job is marked failed. Defaults to `1`.
</ParamField>

Returns `Promise<ZilMateJob>`.

---

### `listJobs(options?)`

Returns an array of all job records, optionally filtered by status or schedule.

Returns `Promise<ZilMateJob[]>`.

---

### `getJob(id)`

Fetches a single job record by its ID.

<ParamField body="id" type="string" required>
  The unique job ID returned by `createJob()`.
</ParamField>

Returns `Promise<ZilMateJob>`.

---

### `getJobLogs(id)`

Retrieves the execution log entries for a specific job.

<ParamField body="id" type="string" required>
  The unique job ID.
</ParamField>

Returns `Promise<JobLog[]>`.

---

### `runJob(id)`

Executes a specific job immediately, regardless of its schedule.

<ParamField body="id" type="string" required>
  The unique job ID to execute right now.
</ParamField>

---

### `runDueJobs()`

Scans all scheduled jobs and executes any whose cron schedule is currently due. Embed this call in a serverless handler or cron endpoint to drive scheduled execution without the CLI worker.

---

### `cancelJob(id)`

Cancels a pending job, preventing future execution.

<ParamField body="id" type="string" required>
  The unique job ID to cancel.
</ParamField>

---

## Types

### `ZilMateJob`

<ResponseField name="id" type="string">
  Unique identifier for the job.
</ResponseField>

<ResponseField name="task" type="string">
  The natural-language instruction the agent executes.
</ResponseField>

<ResponseField name="status" type='"queued" | "running" | "succeeded" | "failed" | "cancelled"'>
  The current lifecycle state of the job.
</ResponseField>

<ResponseField name="schedule" type="string">
  The cron expression defining when the job runs, if any.
</ResponseField>

<ResponseField name="maxAttempts" type="number">
  The maximum number of retry attempts configured for this job.
</ResponseField>

<ResponseField name="attempts" type="number">
  The number of execution attempts made so far.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of when the job was created.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of when the job record was last updated.
</ResponseField>

<ResponseField name="runAt" type="string">
  ISO 8601 timestamp of when the job is scheduled to run next, if set explicitly.
</ResponseField>

<ResponseField name="nextRunAt" type="string">
  ISO 8601 timestamp of the next scheduled execution derived from the cron expression.
</ResponseField>

<ResponseField name="lastRunAt" type="string">
  ISO 8601 timestamp of the most recent execution attempt.
</ResponseField>

<ResponseField name="completedAt" type="string">
  ISO 8601 timestamp of when the job reached a terminal state (`succeeded` or `failed`).
</ResponseField>

<ResponseField name="result" type="string">
  The agent's output text from the most recent successful execution, if available.
</ResponseField>

<ResponseField name="error" type="string">
  The error message from the most recent failed execution, if available.
</ResponseField>

<ResponseField name="qstashScheduleId" type="string">
  The QStash schedule ID assigned when the job is registered for cloud-hosted cron delivery.
</ResponseField>

### `JobLog`

<ResponseField name="id" type="string">
  Unique identifier for this log entry.
</ResponseField>

<ResponseField name="jobId" type="string">
  The ID of the job this log entry belongs to.

<ResponseField name="message" type="string">

<ResponseField name="level" type='"info" | "progress" | "error" | "result"'>

</ResponseField>

## Quick example

Create a weekly summary job, then retrieve its logs after it runs:

```ts
import { createZilMate } from 'zilmate/server';

const zilmate = createZilMate({ sessionId: 'my-session' });

// Create a job that runs every Monday at 9am
const job = await zilmate.createJob({
  task: 'Weekly summary of GitHub PRs and open issues',
  schedule: '0 9 * * 1',
  maxAttempts: 3,
});

console.log('Job created:', job.id, '— status:', job.status); // 'queued'

// After the job runs, retrieve its logs
const logs = await zilmate.getJobLogs(job.id);
logs.forEach((log) => console.log(`[${log.level}] ${log.message}`));
```

<Note>
  Start the local worker in a separate terminal with `zilmate jobs worker` to process the job when its schedule is due.
</Note>
