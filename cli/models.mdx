ZilMate routes different task types to different agent targets, and each target can use a different AI model. `zilmate models` opens an interactive model browser that lists every model available from your AI Gateway, lets you assign a model to an agent target, and persists your selections so they apply automatically on every future run.

---

## `zilmate models`

Open the interactive model browser:

```bash
zilmate models
```

Use the filter flags to narrow the list when you know which provider you want:

```bash
# Show only Anthropic models
zilmate models --provider anthropic

# Show only Google / Gemini models
zilmate models --provider gemini

# Show OpenAI models, 50 per page
zilmate models --provider openai --limit 50

# Jump to page 2 of the full catalogue
zilmate models --page 2
```

**Flags:**

| Flag | Description |
|---|---|
| `-p, --provider <provider>` | Filter by provider or text: `openai`, `google`, `gemini`, `anthropic`, etc. |
| `-l, --limit <number>` | Models per page (default: `20`) |
| `--page <number>` | Page number (default: `1`) |

## Agent key → environment variable mapping

When you select a model in the browser, you choose an agent target. The selection is saved to `~/Downloads/ZilMate/config/models.json` (or the path set by `ZILMATE_WORKSPACE`) and applied on the next run. You can also set any of these variables directly in your `.env` to override the saved selection without opening the browser.

### Core agent targets

| Agent key | Environment variable | Description | Default |
|---|---|---|---|
| `manager` | `ZILO_MANAGER_MODEL` | Main orchestrator, chat, and research agent | `minimax/minimax-m3` |
| `help` | `ZILO_HELP_MODEL` | Quick help and troubleshooting agent | `ZILO_MANAGER_MODEL` fallback |
| `post` | `ZILO_POST_MODEL` | Social copy and caption generation | `ZILO_MANAGER_MODEL` fallback |
| `coding` | `ZILO_CODING_MODEL` | Coding agent, app builder, and QA integration | `ZILO_MANAGER_MODEL` fallback |

### Image generation targets

| Agent key | Environment variable | Description | Default |
|---|---|---|---|
| `image` (provider) | `ZILO_IMAGE_DEFAULT_PROVIDER` | Default image provider: `openai` or `gemini` | `openai` |
| `image` (OpenAI model) | `ZILO_IMAGE_OPENAI_MODEL` | OpenAI image model | `openai/gpt-image-2` |
| `image` (Gemini model) | `ZILO_IMAGE_GEMINI_MODEL` | Gemini image model | `google/gemini-3.1-flash-image` |

### Vision and desktop targets

| Agent key | Environment variable | Description | Default |
|---|---|---|---|
| `screenshotVision` | `ZILMATE_SCREENSHOT_MODEL` | Vision model for screenshot and camera analysis | `google/gemini-3.1-flash-lite` |

### Swarm department targets

Each department in the Digital Corporation swarm can use its own model. All departments fall back to `ZILO_MANAGER_MODEL` when the department-specific variable is not set.

| Department | Environment variable |
|---|---|
| Strategy & Planning | `ZILMATE_DEPT_STRATEGY_MODEL` |
| Software Engineering | `ZILMATE_DEPT_ENGINEERING_MODEL` |
| Growth & Marketing | `ZILMATE_DEPT_GROWTH_MODEL` |
| General Operations | `ZILMATE_DEPT_OPERATIONS_MODEL` |
| Data Analytics | `ZILMATE_DEPT_DATA_MODEL` |
| Security & Audits | `ZILMATE_DEPT_SECURITY_MODEL` |
| Revenue & Finance | `ZILMATE_DEPT_REVENUE_MODEL` |
| Development (internal) | `ZILMATE_DEPT_DEVELOPMENT_MODEL` |

## Inside an interactive session

Inside a `zilmate talk` session, type `/model pick` to open the same model browser without leaving the session:

```
/model pick
```

Changes take effect for the remainder of your current session and are also saved to the workspace config file.

---

## `.env` example

You can set all model overrides directly in `.env` without using the browser:

```env
ZILO_MANAGER_MODEL=anthropic/claude-sonnet-4
ZILO_HELP_MODEL=alibaba/qwen3.7-plus
ZILO_POST_MODEL=alibaba/qwen3.7-plus
ZILO_CODING_MODEL=openai/gpt-4o
ZILO_IMAGE_DEFAULT_PROVIDER=openai
ZILO_IMAGE_OPENAI_MODEL=openai/gpt-image-2
ZILO_IMAGE_GEMINI_MODEL=google/gemini-3-pro-image
ZILMATE_SCREENSHOT_MODEL=google/gemini-3.1-flash-lite
ZILMATE_DEPT_SECURITY_MODEL=anthropic/claude-sonnet-4
ZILMATE_DEPT_ENGINEERING_MODEL=openai/gpt-4o
```

## SDK equivalent

Apply and persist model selections programmatically:

```ts
import { applyStoredModelSelections, saveModelSelection } from 'zilmate/server';

// Load any saved selections before running tasks
await applyStoredModelSelections();

// Persist new selections to workspace config
await saveModelSelection('manager', 'anthropic/claude-sonnet-4');
await saveModelSelection('coding', 'openai/gpt-4o');
await saveModelSelection('help', 'alibaba/qwen3.7-plus');
```

<Tip>
Match models to task requirements. Use a fast, low-latency model for `help` — where response speed matters most — and a more capable reasoning model for `manager` and `coding`. The defaults in `.env` are a good starting point; tune from there as your usage patterns become clear.
</Tip>
