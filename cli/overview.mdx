The `zilmate` CLI is the fastest way to interact with ZilMate agents from your terminal. Run one-shot tasks, start persistent interactive sessions, and manage memory and background jobs — all without writing a single line of code. Every capability available in the SDK is reachable from the command line.

## Installation

Install the CLI globally with npm:

```bash
npm install -g zilmate
```

After installation, run the setup wizard to configure your credentials:

```bash
zilmate setup
```

To see all available commands at any time:

```bash
zilmate --help
```

<Note>
Running `zilmate` with no arguments opens the status dashboard and guided launcher menu. If the AI Gateway key is missing, the launcher walks you through setup before starting a chat session.
</Note>

## Command Overview

The table below lists every top-level command group and what it does.

| Command | Description |
|---|---|
| `zilmate setup` | Interactive configuration wizard — set API keys, voice, jobs, and more |
| `zilmate doctor` | Health-check all configured services and report OK / WARN / ERROR |
| `zilmate talk` | Start a persistent interactive chat session with the Manager agent |
| `zilmate manager "<task>"` | Run a one-shot task through the Manager agent and exit |
| `zilmate swarm "<goal>"` | Launch a full Digital Corporation multi-agent swarm workflow |
| `zilmate jobs create/worker/list/run/logs/cancel/listen` | Create jobs, run the local worker, list history, and webhook-test with tunnels |
| `zilmate remember/recall/forget` | Store facts, search memories, and delete entries |
| `zilmate memory list` | List all stored memory entries |
| `zilmate models` | Browse available AI models and assign them to agent targets |
| `zilmate triggers listen` | Stream real-time Composio trigger events to the terminal |
| `zilmate update` | Update the global CLI to the latest published version |

## CLI ↔ SDK Parity

Every CLI command maps to an SDK method you can call in Next.js routes, server workers, or automation scripts.

| CLI | SDK equivalent |
|---|---|
| `zilmate talk` | `createZilMate().manager({ message })` |
| `zilmate manager "<task>"` | `createZilMate().manager({ message })` |
| `zilmate swarm "<goal>"` | Manager with swarm routing via `createZilMate().manager()` |
| `zilmate remember "<text>"` | `zilmate.remember({ text, tags })` |
| `zilmate recall "<query>"` | `zilmate.recall({ query, limit })` |
| `zilmate jobs create "<task>"` | `zilmate.createJob({ task, schedule })` |
| `zilmate jobs worker` | `zilmate.runDueJobs()` |
| `/model pick` (in `talk`) | `saveModelSelection('manager', 'model-id')` |
