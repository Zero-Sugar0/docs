ZilMate's job system lets you queue AI tasks to run on a schedule or on demand — without keeping a chat session open. Jobs run locally via the `worker` command, or remotely via Upstash QStash when you need tasks to execute even when your machine is asleep. All job subcommands are available under `zilmate jobs`.

<Note>
Set `ZILMATE_JOBS_ENABLED=true` in your `.env` before using any `jobs` subcommand. Run `zilmate setup` to configure this interactively.
</Note>

---

## Subcommands

### `zilmate jobs create`

Queue a new background job. Pass the task description as the first argument.

```bash
zilmate jobs create "<task description>"
```

**Examples:**

```bash
# One-time job with no schedule
zilmate jobs create "Research today's priority updates and summarize them"

# Daily recurring job
zilmate jobs create "Prepare my weekday morning briefing" --schedule "daily"

# Hourly job
zilmate jobs create "Check for new GitHub notifications and log them" --schedule "hourly"

# Cron expression
zilmate jobs create "Weekly revenue report" --schedule "cron:0 9 * * 1"

# Run at a specific ISO 8601 time
zilmate jobs create "Deploy staging to production" --schedule "daily" --run-at "2025-08-01T09:00:00Z"
```

**Flags:**

| Flag | Description |
|---|---|
| `--schedule <schedule>` | Schedule expression: `hourly`, `daily`, `every 15 minutes`, or `cron:* * * * *` |
| `--run-at <date>` | ISO 8601 date/time for the first run |

---

### `zilmate jobs worker`

Start the local job processor. The worker polls for due jobs, executes them one by one, and logs each result.

```bash
zilmate jobs worker
```

The worker keeps running until you stop it with `Ctrl+C`. Use `--once` to process all currently due jobs and then exit — useful in scripts and CI:

```bash
zilmate jobs worker --once
```

**Flags:**

| Flag | Description |
|---|---|
| `-i, --interval <seconds>` | Polling interval in seconds (default: `10`) |
| `--once` | Process due jobs once and exit |
| `--quiet` | Suppress worker status messages |

---

### `zilmate jobs list`

Display all jobs with their status, last run time, and next scheduled run.

```bash
zilmate jobs list
```

Filter by status or limit the output:

```bash
zilmate jobs list --status queued
zilmate jobs list --status failed
zilmate jobs list --limit 50
```

**Flags:**

| Flag | Description |
|---|---|
| `--status <status>` | Filter by `queued`, `running`, `succeeded`, `failed`, or `cancelled` |
| `-l, --limit <number>` | Maximum jobs to return (default: `25`) |

---

### `zilmate jobs status <id>`

Show the full details for a single job by ID.

```bash
zilmate jobs status job_abc123
```

---

### `zilmate jobs run <id>`

Execute a specific job immediately, regardless of its schedule.

```bash
zilmate jobs run job_abc123
```

Use this to test a job right after creating it, or to re-run a failed job without waiting for the next scheduled window.

---

### `zilmate jobs logs <id>`

Print execution logs for a specific job.

```bash
zilmate jobs logs job_abc123
```

Logs include the agent's output, any tool calls made, and the final result or error message.

---

### `zilmate jobs cancel <id>`

Cancel a pending job so it does not execute on its next scheduled run.

```bash
zilmate jobs cancel job_abc123
```

---

### `zilmate jobs listen`

Start a local webhook server that receives job triggers from QStash. Use this during local development to test hosted schedule delivery without deploying a public endpoint.

```bash
# Start the webhook server on the default port (8787)
zilmate jobs listen

# Start with a Cloudflare quick tunnel to expose a public HTTPS URL automatically
zilmate jobs listen --tunnel
```

**Flags:**

| Flag | Description |
|---|---|
| `-p, --port <number>` | Local webhook port (default: `8787`) |
| `--tunnel` | Start a Cloudflare quick tunnel alongside the server (requires `cloudflared`) |

#### How `--tunnel` works

When you pass `--tunnel`, ZilMate spawns a `cloudflared` quick-tunnel process alongside the local webhook server. The tunnel provides a stable public HTTPS URL that you copy into `ZILMATE_PUBLIC_JOB_WEBHOOK_URL` and into your QStash endpoint configuration. QStash then delivers scheduled job triggers directly to your local machine — without needing a deployed server.

```bash
# Start the listener with a tunnel
zilmate jobs listen --tunnel

# Example output:
# ✅  Webhook server running on http://localhost:8787
# 🌐  Cloudflare tunnel active: https://random-name.trycloudflare.com
# 📋  Set ZILMATE_PUBLIC_JOB_WEBHOOK_URL=https://random-name.trycloudflare.com
```

Copy the tunnel URL into your environment and point QStash at it:

```bash
# Update your .env with the tunnel URL
zilmate setup --yes --job-webhook-url https://random-name.trycloudflare.com
```

<Tip>
Run `zilmate setup --install-cloudflare-deps true` if `cloudflared` is not yet installed. ZilMate downloads the correct platform binary automatically.
</Tip>

## End-to-end example

```bash
# 1. Create a daily briefing job
zilmate jobs create "Pull my open GitHub PRs, Notion tasks, and Slack DMs and write a morning briefing" \
  --schedule "daily" \
  --run-at "2025-08-01T08:00:00Z"

# 2. Confirm it was queued
zilmate jobs list --status queued

# 3. Run it immediately to verify the output
zilmate jobs run job_abc123

# 4. Review the execution output
zilmate jobs logs job_abc123

# 5. Start the local worker to process future scheduled runs
zilmate jobs worker

# 6. Cancel the job if you no longer need it
zilmate jobs cancel job_abc123
```

## SDK equivalent

The jobs system is fully accessible from the SDK for use in Next.js routes and server workers:

```ts
import { createZilMate } from 'zilmate/server';

const zilmate = createZilMate({ sessionId: 'worker-session' });

// Create a scheduled job
const job = await zilmate.createJob({
  task: 'Weekly summary of GitHub PRs merged this week',
  schedule: '0 9 * * 1',
});

// Execute any due jobs (use this in a background worker process)
await zilmate.runDueJobs();

// Expose a webhook route for QStash to call
// app/api/jobs/webhook/route.ts
export async function POST(req: Request) {
  const body = await req.json();
  const result = await zilmate.handleJobWebhook(body, process.env.ZILMATE_JOB_WEBHOOK_SECRET);
  return Response.json({ ok: true, jobId: result.id });
}
```

| CLI | SDK |
|---|---|
| `zilmate jobs create "<task>"` | `zilmate.createJob({ task, schedule })` |
| `zilmate jobs worker` | `zilmate.runDueJobs()` |
| `zilmate jobs listen` | Webhook route + `zilmate.handleJobWebhook()` |
